Why Is Sketching a User Interface a Good Idea?

Sketching the user interface (UI) before development can be very helpful, especially for complex applications like an electric scooter rental app. It allows your team and stakeholders to quickly understand how the interface might look and function, without needing long explanations.

Sketching can also reveal potential issues in the user experience (UX) early, so they can be fixed before any code is written. This approach encourages you to think through the user’s journey in the app, leading to a design that’s more focused on their needs.

How to Choose a Tool for Sketching UI

There are many tools available for sketching user interfaces, and the right choice depends on a few factors:

  • Your experience with visualizing ideas and arranging elements like shapes and text
  • Your understanding of graphic design concepts
  • Your timeframe for completing the sketch

You can start by searching for popular sketching tools and exploring their features. In this report, I will be using two free web-based platforms, Figma and MockFlow (both offer free plans with some limitations).

  • Figma: A widely used tool for sketching and prototyping UIs; it is considered the industry standard.
  • MockFlow: An intuitive, drag-and-drop tool for creating UI designs quickly and easily.

Prerequisites

  • I assume that you have created an account on the following services: Figma and MockFlow.
  • I will use one requirement specification translated to English from Kravspecification Svenska Elsparkcyklar AB (2024). The requirement is listed below. I have broken it down to fit the goal of sketching a UI.

Customer Web Interface

  • The customer can view a history of their trips, with all saved details, including the cost.
    • Display a list of trips with relevant data and the total cost.
      • Date field: Shows the trip’s date, month, and year (22 July 2024).
      • Total cost: Displays the amount and currency (SEK 299.00).

Interface Specifications:

  • The UI will be sketched for mobile phones with dimensions 390 x 844.
  • A common navigation bar will be added across all pages.

Sketching UI Using MockFlow

Sketch Navigation Menu

  1. Add an iOS Frame:
    • Select Elements > Frames > iOS Frame Light from the left menu.
    • Drag and drop the iOS Frame onto the canvas and set its dimensions to 390 x 844.
  1. Add Text Labels:
    • From the left menu, select Elements > Text > Label and drag it onto the canvas.
    • Double-click the label to change its size, text content, text alignment and add an icon.
    • To add an icon to the label, select Icons > Select icon from the left menu and choose an icon.
  1. Add more Text Labels:
    • Repeat the Add and Edit label actions to add more ‘buttons’. I added labels with the text ‘Skanna’ and ‘Profile’. Do not forget icons; I chose ‘keyboard’ and ‘user’ icons.
    • Select all the labels by holding Shift while clicking each one or dragging a selection box.
    • Use options Align > Align Middle and Distribute > Distribute Horizontal Space to align them properly.
  1. Place Navigation Bar:
    • Select and place the navigation bar inside the iOS Frame.

Sketch Customer View Over Trip History

  1. Add Text and Currency Labels:
    • Use Text > Label to display the date, month, year, and total cost (e.g., "12 Mars 14:08 SEK 123.00").
    • Select an icon for the amount label to indicate user actions.
    • Arrange the text and icons inside the iOS Frame and align them properly.
  1. Enhance Design:
    • Add a horizontal line Elements > Shapes > Horizontal Line beneath each trip item for better visual appeal.
